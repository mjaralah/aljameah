## المشكلة

أقسام الصفحة الرئيسية موجودة في جدول `page_content` لكن حقول `title` و `content` و `data` فارغة (NULL). لذلك في لوحة التحكم → "محتوى الصفحات" → تبويب "الصفحة الرئيسية" تظهر الأقسام بحقول فارغة، بينما النصوص المعروضة فعلياً على الموقع تأتي من ملفات الترجمة (`src/i18n/ar.ts` و `en.ts`) كقيم احتياطية، فلا يمكن تعديلها أو حذفها أو إخفاؤها من لوحة التحكم.

إضافة لذلك، محرر بعض الأقسام في لوحة التحكم (مثل **شبكة البرامج** و **آخر الأخبار**) لا يوفّر حقل "العنوان الفرعي (Eyebrow)" رغم استخدامه في الواجهة.

## الحل

### 1) تعبئة الأقسام الحالية بالنصوص الافتراضية (بيانات فقط، بدون تغيير بنية الجدول)
سأكتب أمر تحديث (UPDATE) لكل قسم في `page_key='home'` يملأ `title` / `content` / `data` بالقيم العربية الافتراضية الحالية التي تظهر الآن على الموقع:

| القسم | الحقول التي ستُملأ |
|---|---|
| `stats` | `data.items` = الإحصائيات الأربع (مستفيدين، برامج، متطوعين، سنوات) مع الأيقونات والأرقام الحالية |
| `about_preview` | `title` = "الجمعية الخيرية" • `content` = النبذة الحالية • `data.cta_label` = "اعرف أكثر" • `data.cta_url` = "/about" |
| `programs` | `title` = "برامجنا وخدماتنا" • `data.eyebrow` = "ما نقدّمه" |
| `satisfaction` | `title` = "مؤشرات رضا المستفيدين" • `data.cta_label` = "عرض نتائج الاستبيانات" • `data.cta_url` = "/surveys" |
| `news` | `title` = "آخر الأخبار" • `data.eyebrow` = "الأخبار" |
| `partners` | `title` = "شركاء النجاح" • `data.eyebrow` = "شركاؤنا" |
| `volunteer_cta` | `title` = "كن متطوعاً معنا" • `content` = نص الدعوة الحالي • `data.cta_label` = "تطوع الآن" • `data.cta_url` = "/e-services/volunteer" |

نتيجة هذه الخطوة: ستظهر النصوص الحالية داخل لوحة التحكم فوراً، ويمكن تعديلها أو إفراغها أو إخفاء القسم كلياً عبر زر النشر، أو حذفه نهائياً — كل ذلك يعمل أصلاً بالكود الحالي بمجرد وجود البيانات.

(القيم الاحتياطية في ملفات `i18n` ستبقى كما هي، فقط كاحتياط إذا فُرّغ الحقل في قاعدة البيانات. هذا يحافظ على السلامة ولا يكسر اللغة الإنجليزية.)

### 2) إضافة محرّر "العنوان الفرعي (Eyebrow)" لقسمَي `programs` و `news` في `src/pages/admin/AdminPageContentPage.tsx`
داخل دالة `renderStructured`، إضافة فرع جديد:

```tsx
if (s.section_key === "programs" || s.section_key === "news") {
  const d = s.data || {};
  return (
    <div>
      <Label>العنوان الفرعي (Eyebrow)</Label>
      <Input value={d.eyebrow ?? ""}
        onChange={(e) => updateData(s.id, "eyebrow", e.target.value)} />
    </div>
  );
}
```

بذلك يحصل كل قسم على المحرّر المناسب لكل ما يظهر على الموقع.

### تفاصيل تقنية

- **ملف معدَّل:** `src/pages/admin/AdminPageContentPage.tsx` (إضافة فرع إديتر صغير فقط).
- **تعديل بيانات:** أمر `UPDATE` واحد على جدول `page_content` للصفحة `home` (يستخدم `COALESCE` للحفاظ على أي قيم أدخلها المستخدم سابقاً ولا يدوسها).
- **بدون تغيير schema** ولا تعديل على مكوّنات الواجهة العامة (Hero/News/...): هي تقرأ من `page_content` أصلاً وتسقط على i18n عند الفراغ — أي سلوك سيبقى كما هو، فقط ستصبح النصوص قابلة للتحرير من لوحة التحكم.
- شريط العرض الرئيسي (Hero) له جدول مستقل `hero_slides` وله رابط سريع موجود مسبقاً في تبويب الصفحة الرئيسية — لا حاجة لتعديل.
