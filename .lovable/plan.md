## الهدف

تحويل التذييل من محتوى ثابت في الكود إلى محتوى قابل للتحرير بالكامل من لوحة التحكم، مع دعم:
- تعديل نص تعريف الجمعية (الاسم، الوصف، الشعار الصغير).
- إدارة كاملة (إضافة/تعديل/حذف/إعادة ترتيب) لروابط عمودَي **روابط سريعة** و**الخدمات الإلكترونية**.
- إعادة ترتيب أعمدة التذييل ككل (تعريف، روابط سريعة، خدمات، قانونية، تواصل).
- دعم اللغتين العربية والإنجليزية لكل عنصر.

## البنية المقترحة في قاعدة البيانات

### 1) جدول `footer_sections` — لإدارة الأعمدة وترتيبها
| الحقل | الوصف |
|---|---|
| `id` | معرف |
| `section_key` | `brand` \| `quick` \| `eservices` \| `legal` \| `contact` (ثابت — لا يُحذف) |
| `title_ar` / `title_en` | عنوان العمود (يُستخدم للأعمدة ذات القوائم) |
| `sort_order` | ترتيب الظهور |
| `published` | إظهار/إخفاء (يحلّ محل مفاتيح `footer_*` الحالية في `pages_visibility`) |

سيتم زرع 5 صفوف افتراضية مطابقة للوضع الحالي.

### 2) جدول `footer_links` — لروابط عمودَي "روابط سريعة" و"الخدمات الإلكترونية"
| الحقل | الوصف |
|---|---|
| `id` | معرف |
| `section_key` | `quick` أو `eservices` |
| `label_ar` / `label_en` | نص الرابط |
| `url` | المسار (داخلي مثل `/about` أو خارجي) |
| `sort_order` | الترتيب داخل العمود |
| `published` | إظهار/إخفاء الرابط |

سيتم زرع الروابط الحالية افتراضياً.

### 3) أعمدة جديدة في `site_settings` لقسم تعريف الجمعية
- `footer_brand_name_ar` / `footer_brand_name_en`
- `footer_brand_tagline_ar` / `footer_brand_tagline_en`
- `footer_brand_about_ar` / `footer_brand_about_en`

(تترك فارغة افتراضياً → يستخدم التذييل القيم من ملفات الترجمة كـ fallback، فلا يتغير الشكل الحالي.)

سياسات RLS: قراءة عامة، تعديل للطاقم فقط (نفس نمط بقية الجداول).

## واجهة لوحة التحكم

صفحة جديدة **"تذييل الموقع"** (`/admin/footer`) فيها 3 أقسام:

1. **تعريف الجمعية** — حقول الاسم/الشعار/الوصف بالعربية والإنجليزية + حقل ملاحظة "اتركه فارغاً لاستخدام النص الافتراضي".
2. **أعمدة التذييل** — قائمة الأعمدة الخمسة مع:
   - مقبض سحب لإعادة الترتيب (باستخدام `SortableList`).
   - مفتاح إظهار/إخفاء.
   - تعديل العنوان بالعربية والإنجليزية.
3. **روابط الأعمدة** — تبويبان (روابط سريعة / خدمات إلكترونية)، كل تبويب يعرض جدول الروابط مع:
   - إضافة رابط جديد (نص ar/en + URL).
   - تعديل.
   - حذف.
   - إعادة ترتيب بالسحب.
   - تبديل النشر.

يُضاف رابط الصفحة في القائمة الجانبية ضمن قسم "محتوى الصفحات".

## التغييرات في الواجهة العامة

- إنشاء hooks جديدة: `useFooterSections()` و `useFooterLinks()`.
- إعادة كتابة `src/components/layout/Footer.tsx` ليبني الأعمدة ديناميكياً من قاعدة البيانات حسب `sort_order`، مع استخدام بيانات `site_settings` لقسم العلامة (مع fallback لملفات الترجمة).
- إزالة مفاتيح `footer_brand` / `footer_quick` / `footer_eservices` / `footer_legal` / `footer_contact` / `footer_bottom` من قائمة `PUBLIC_PAGES` لأن التحكم انتقل لصفحة التذييل المخصصة (مع الإبقاء على `footer_bottom` كمفتاح بسيط).

## نقاط مهمة قبل البدء

- شريط الحقوق السفلي (`© 2026 ...`) سيبقى مرتبطاً بمفتاح `footer_bottom` الحالي وقابلاً للإظهار/الإخفاء فقط (لا يحتاج تحرير نص حالياً، يمكن إضافته لاحقاً إن طُلب).
- عمود "معلومات التواصل" يستخدم بالفعل الحقول الموجودة في الإعدادات (هاتف/بريد/عنوان) — سيُكتفى بذلك دون تكرار.
- لن يُكسر أي محتوى موجود: الـ fallback لملفات الترجمة يضمن أن المظهر يبقى مطابقاً قبل أي تعديل.
