# تحليل المقارنة بين النسخة الأصلية والنسخة الحالية

## النتيجة الرئيسية

بعد مقارنة **كود المشروع** و**هيكل قاعدة البيانات** و**ملفات الترحيل (Migrations)** بين النسختين:

- ✅ **جميع ملفات الكود متطابقة 100%**: نفس الصفحات، نفس مكونات لوحة التحكم، نفس مكونات الواجهة، نفس مكونات `about/sections` و `admin/about` و `admin/blocks`.
- ✅ **جميع ملفات الـ Migrations متطابقة (20 ملف)** — هيكل قاعدة البيانات (الجداول والأعمدة) سليم بالكامل.
- ❌ **قاعدة البيانات فارغة تماماً** — هذا هو السبب الفعلي وراء اختفاء "التعديلات".

## ما الذي اختفى فعلياً؟ (محتوى مفقود وليس ميزات)

عند عمل Remix، يتم نسخ الكود وهيكل الجداول فقط، ولا يتم نسخ الصفوف (Rows). الجداول التالية كلها فارغة (0 صف):

| الجدول | المحتوى المفقود |
|---|---|
| `site_settings` | إعدادات الموقع العامة (الشعار، الألوان، التواصل، إعدادات أداة التقييم) |
| `hero_slides` | شرائح البطل في الصفحة الرئيسية |
| `page_content` | أقسام الصفحة الرئيسية القابلة للتحرير (بطل/برامج/أخبار/شركاء) |
| `about_content` | كل أقسام صفحة "من نحن" بما فيها الهيكل التنظيمي، شهادة التسجيل، أعضاء الجمعية العمومية (`assembly_members`) |
| `programs` | البرامج |
| `news` | الأخبار |
| `partners` | الشركاء |
| `board_members` + `board_settings` | مجلس الإدارة |
| `governance_categories` + `governance_documents` | وثائق الحوكمة |
| `legal_pages` | الصفحات القانونية (الخصوصية، الشروط...) |
| `custom_pages` | الصفحات المخصصة المُنشأة عبر Page Builder |
| `custom_forms` | النماذج المخصصة |
| `surveys` + `survey_questions` | الاستبيانات |

> ملاحظة: الميزات نفسها (محرر النصوص الغني، منتقي الأيقونات، Block Builder، السحب والإفلات، assembly_members، تفاصيل الأخبار، ScrollToTop، أوضاع العرض للشهادة والهيكل... إلخ) **موجودة كلها في الكود**. هي تبدو مختفية فقط لأن لا توجد بيانات لعرضها.

## خيارات الحل

أحتاج قراراً منك قبل التنفيذ — اختر أحد المسارات:

### الخيار أ: إعادة إدخال يدوي عبر لوحة التحكم (الأنظف)
- إنشاء حساب مدير جديد، ثم الدخول إلى `/admin` وإدخال المحتوى الفعلي الذي تريده.
- ميزة: محتوى حقيقي يخص جمعيتك من البداية.
- عيب: يستهلك وقتاً.

### الخيار ب: تشغيل Seed افتراضي
- أكتب migration واحد يملأ الجداول بمحتوى تجريبي افتراضي (أقسام "من نحن" الأساسية، أمثلة أخبار/برامج/شركاء، أقسام الصفحة الرئيسية، إعدادات الموقع الافتراضية) — يطابق نفس الهيكل الذي كان بُني سابقاً.
- ميزة: الموقع يعود "حياً" خلال دقائق ويمكنك بعدها تعديل المحتوى من اللوحة.
- عيب: محتوى عام/تجريبي يحتاج تخصيصاً لاحقاً.

### الخيار ج: ترحيل البيانات الفعلية من النسخة الأصلية
- إذا كان لديك وصول للوحة تحكم النسخة الأصلية، يمكنك تصدير الجداول من قسم **Cloud → Database → Tables → Export** ثم إرسالها لي لاستيرادها هنا.
- ميزة: استعادة بيانات حقيقية 1:1.
- عيب: يتطلب خطوة يدوية منك للتصدير.

## ما الذي سأفعله بعد قرارك

- **لو اخترت (أ)**: سأتأكد فقط من وجود حساب مدير ابتدائي ومن أن جميع صفحات لوحة التحكم تعمل بدون أخطاء عند الفراغ، ثم تتولى الإدخال بنفسك.
- **لو اخترت (ب)**: سأنشئ ملف seed SQL واحد يُعيد المحتوى الافتراضي لكل الجداول المذكورة أعلاه.
- **لو اخترت (ج)**: انتظر منك ملفات التصدير (CSV لكل جدول) وسأكتب سكربت استيراد.
