# تحسين إدارة صفحة "من نحن"

ملاحظتان من المستخدم تم تأكيدهما من الشيفرة:

1. أقسام صفحة "من نحن" في لوحة التحكم (`/admin/about`) ليست قابلة للسحب والإفلات لإعادة الترتيب، رغم أن جدول `about_content` يحتوي على عمود `sort_order` يستخدمه العرض العام.
2. قسما **الهيكل التنظيمي** (`structure`) و**شهادة التسجيل** (`registration`) في صفحة `/about` مكتوبان كقيم ثابتة (hardcoded) داخل `src/pages/About.tsx`، ولا يوجد لهما أي مدخل في `KEY_LABELS` ولا واجهة تحرير في `AdminAboutPage`.

## ما سيتم تنفيذه

### 1) سحب وإفلات لإعادة ترتيب الأقسام
- إضافة `SortableList` + `SortableItem` (الموجودان مسبقاً في `src/components/admin/SortableList.tsx`) حول قائمة بطاقات الأقسام في `AdminAboutPage`.
- مقبض سحب (GripVertical) في رأس كل بطاقة.
- عند الإفلات: استدعاء `persistSortOrder(supabase, "about_content", newIds)` ثم إعادة التحميل.
- صفحة `/about` ستعرض الأقسام وفق `sort_order` تلقائياً (الـ hook `useAboutContent` يرتب بـ sort_order).

### 2) تحويل قسم "الهيكل التنظيمي" إلى محتوى قابل للتحرير
- إضافة `structure` إلى `KEY_LABELS` وإلى renderer هيكلي جديد في `AdminAboutPage`.
- شكل البيانات في `data` jsonb:
  ```text
  {
    "intro": "نص تمهيدي",
    "nodes": [
      { "title": "الجمعية العمومية", "subtitle": "السلطة العليا" },
      { "title": "مجلس الإدارة", "subtitle": "الإشراف والحوكمة" },
      { "title": "المدير التنفيذي", "subtitle": "القيادة التنفيذية" }
    ],
    "departments": [
      { "title": "إدارة البرامج", "desc": "..." },
      ...
    ]
  }
  ```
- واجهة التحرير: حقل نص تمهيدي + قائمتان (عُقد هرمية + إدارات) كل منهما قابلة للإضافة/الحذف.
- صفحة `/about` تقرأ من `aboutMap["structure"]` بدلاً من الثوابت، مع fallback للقيم الحالية.

### 3) تحويل قسم "شهادة التسجيل" إلى محتوى قابل للتحرير
- إضافة `registration` إلى `KEY_LABELS` وrenderer هيكلي جديد.
- شكل البيانات:
  ```text
  {
    "badge_label": "جهة مرخّصة وموثّقة",
    "heading": "شهادة تسجيل سارية المفعول",
    "rows": [
      { "label": "رقم التسجيل", "value": "1234/2020" },
      ...
    ],
    "pdf_url": "..."
  }
  ```
- واجهة التحرير: حقول العنوان + قائمة الصفوف (label/value) + رابط/رفع ملف الـ PDF عبر `MediaUpload`.
- زر "تحميل نسخة الشهادة" في `/about` يصبح فعّالاً (يفتح `pdf_url`) ويختفي إن لم يكن موجوداً.

### 4) ترحيل (migration) لإضافة الصفّين الجديدين
- إدراج صفّين في `about_content` بـ `section_key = 'structure'` و `'registration'` ببيانات تطابق المحتوى الظاهر حالياً، حتى لا يتغيّر مظهر الصفحة قبل التحرير.

## التفاصيل التقنية

- **الملفات المعدّلة**:
  - `src/pages/admin/AdminAboutPage.tsx` — إضافة DnD + حالتي structure/registration.
  - `src/pages/About.tsx` — قراءة `structure` و `registration` من `aboutMap` مع fallback.
- **الملفات المضافة**: لا شيء (نستخدم `SortableList` و `MediaUpload` الموجودين).
- **قاعدة البيانات**: إدراج صفّين فقط في `about_content`. لا تغيير في المخطط أو RLS.
- **هوية الموقع**: تصميم القسمين في `/about` لن يتغيّر بصرياً — فقط مصدر النصوص.

## أثر تجربة المستخدم

- المسؤول يستطيع إعادة ترتيب أقسام "من نحن" بالسحب، كما هو متاح في باقي صفحات لوحة التحكم.
- جميع المحتوى الظاهر في `/about` يصبح قابلاً للتحرير من لوحة التحكم دون استثناء.
