# عرض قسمَي "الهيكل التنظيمي" و"شهادة التسجيل" كصورة مضمّنة

إضافة خيار في لوحة التحكم لاختيار طريقة عرض كل من قسم الهيكل التنظيمي وقسم شهادة التسجيل في صفحة "من نحن":
- إما المحتوى المنظم الحالي (نقاط/جدول).
- أو صورة واحدة مضمّنة يرفعها المسؤول.

## السلوك المتفق عليه

- **خيار واحد فقط في كل قسم**: عند اختيار "صورة" يختفي المحتوى المنظم تماماً، والعكس صحيح.
- **الهيكل التنظيمي (وضع الصورة)**: عرض الصورة بعرض كامل ضمن حاوية القسم مع الحفاظ على نسبة الأبعاد (object-contain) بدون اقتصاص، وبدون lightbox.
- **شهادة التسجيل (وضع الصورة)**: تختفي صفوف الجدول وتظهر الصورة فقط، مع إبقاء زر "تحميل نسخة الشهادة" إن تم رفع PDF.
- **الامتدادات المسموحة للرفع**: PNG / JPG / WEBP فقط.
- **الموائمة الآلية**: صورة متجاوبة (`max-width: 100%`, `height: auto`, `object-contain`)، بإطار/زوايا منسجمة مع بقية بطاقات الصفحة، وحد أقصى للارتفاع لمنع التمدد المفرط على الشاشات الكبيرة.

## التغييرات

### 1) شكل البيانات في `about_content.data`

قسم `structure`:
```text
{
  "display_mode": "content" | "image",
  "image_url": "...",     // عند image
  "intro": "...",          // عند content
  "nodes": [...],
  "departments": [...]
}
```

قسم `registration`:
```text
{
  "display_mode": "content" | "image",
  "image_url": "...",     // عند image
  "badge_label": "...",
  "heading": "...",
  "rows": [...],
  "pdf_url": "..."
}
```

`display_mode` افتراضياً `content` للحفاظ على المظهر الحالي.

### 2) لوحة التحكم — `src/pages/admin/AdminAboutPage.tsx`

داخل renderer كل قسم منهما:
- إضافة محدّد (Tabs أو RadioGroup) بخيارَين: "محتوى منظم" / "صورة".
- عند اختيار "صورة": إظهار `MediaUpload` (مقيّد بـ png/jpg/webp) لحقل `image_url`، وإخفاء حقول المحتوى المنظم.
- عند اختيار "محتوى منظم": إظهار الحقول الحالية وإخفاء حقل الصورة.
- حفظ القيمة في `data.display_mode`.

### 3) صفحة العرض — `src/pages/About.tsx`

- قسم الهيكل التنظيمي: إذا `display_mode === "image"` و`image_url` موجود، عرض `<img>` بعرض كامل داخل بطاقة القسم بنفس padding/الإطار الحالي، مع `object-contain` و`max-h-[640px]`. وإلا عرض المحتوى المنظم كما هو.
- قسم شهادة التسجيل: إذا `display_mode === "image"` و`image_url` موجود، استبدال صفوف الجدول بصورة الشهادة داخل نفس البطاقة (إطار ناعم، `object-contain`, `max-h-[720px]`)، مع إبقاء البادج/العنوان/زر تنزيل PDF إن وُجد.

### 4) قاعدة البيانات

لا حاجة لأي migration — العمود `data` من نوع jsonb وبنيته مرنة. الصفوف الحالية ستبقى تعمل لأن `display_mode` غير الموجود يُعامَل كـ `content`.

## أثر تجربة المستخدم

- المسؤول يستطيع بنقرة واحدة التبديل بين شرح مفصّل أو صورة جاهزة (مفيد للمنظمات التي لديها مخطط هيكل تنظيمي مصمَّم مسبقاً، أو نسخة مصوّرة من الشهادة).
