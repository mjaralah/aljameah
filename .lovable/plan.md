# خطة بناء المساعد الافتراضي للوحة التحكم

مساعد عائم في نافذة محادثة داخل لوحة التحكم، يساعد المدير غير التقني على إدارة كل أقسام الموقع، **بدون أي اعتماد على نماذج لغوية (GPT/Claude/Gemini)** — مجاني 100% ولا يستهلك أي رصيد ذكاء اصطناعي.

## كيف يعمل بدون ذكاء اصطناعي؟

نعتمد على ثلاث تقنيات مجانية تعمل محلياً في المتصفح:

1. **قاعدة معرفة منظمة (Knowledge Base)** — مقالات/مواضيع إرشادية مخزنة في قاعدة البيانات لكل قسم من أقسام لوحة التحكم.
2. **بحث نصي ذكي (Fuzzy Search)** بواسطة مكتبة `Fuse.js` المجانية — يطابق سؤال المستخدم مع المواضيع حتى لو وُجدت أخطاء إملائية أو صياغة مختلفة.
3. **محرّكات قرار بسيطة (Decision Trees / Wizards)** — أزرار اختيار توجّه المستخدم خطوة بخطوة ("ماذا تريد أن تفعل؟ → إضافة خبر → اختر الصورة... → اضغط هنا").

النتيجة: محادثة تبدو ذكية، فورية، تعمل دون إنترنت خارجي، ودون أي تكلفة.

## مكونات المساعد

### 1. زر عائم (FAB) في كل صفحات `/admin/*`
- زر دائري في الزاوية السفلية مع شارة "؟" وأيقونة محادثة.
- عند الضغط: تفتح نافذة محادثة (`Sheet` أو `Dialog` بحجم 400×600).
- يبقى ظاهراً في كل الصفحات (مدمج في `AdminLayout`).

### 2. نافذة المحادثة
- **شريط علوي**: اسم المساعد + زر إغلاق + زر بحث + زر "ابدأ من جديد".
- **رسالة ترحيب** تعرض الصفحة الحالية ("أنت الآن في: محتوى الصفحات") وتقترح 3-4 أسئلة شائعة لهذه الصفحة.
- **منطقة الرسائل**: فقاعات المستخدم/المساعد + بطاقات إجراء (أزرار).
- **شريط إدخال**: حقل بحث/سؤال + زر إرسال.

### 3. أنواع الرسائل التي يرسلها المساعد
- **نص + خطوات مرقمة** (شرح كيف تضيف خبر).
- **صور/فيديو توضيحي** (لقطة شاشة لكل خطوة).
- **أزرار إجراء مباشرة** ("اذهب إلى صفحة الأخبار" → ينقل المستخدم فوراً عبر `react-router`).
- **اقتراحات متابعة** ("هل تريد معرفة كيفية حذف خبر؟").

### 4. طرق التفاعل الثلاث المدمجة
- **(أ) أسئلة جاهزة**: قائمة فئات (الأخبار، البرامج، الاستبيانات...) ← المستخدم يضغط على الفئة ← قائمة أسئلة ← الجواب.
- **(ب) بحث حر**: المستخدم يكتب سؤال ← Fuse.js يبحث في عنوان+محتوى+كلمات مفتاحية للمواضيع ويعرض أفضل 5 نتائج.
- **(ج) معالج خطوات (Wizard)**: لمهام مركبة ("أريد إنشاء استبيان") ← المساعد يطرح أسئلة متتالية ويوصل المستخدم للنتيجة.

### 5. بحث شامل في محتوى الموقع الفعلي (وليس الشروحات فقط)
- يستطيع المساعد البحث في الأخبار، البرامج، الصفحات المخصصة، الاستبيانات، الأعضاء، الشركاء... ويعرض رابط مباشر لتعديل العنصر.
- مثال: "أين خبر التخريج؟" ← يبحث في `news` ويعرض زر "تعديل هذا الخبر".

## هيكل قاعدة البيانات

جدول جديد واحد فقط:

```text
help_articles
├── id (uuid)
├── category (text)         — مثل: news, programs, surveys, forms, governance...
├── title (text)            — "كيف أضيف خبر جديد"
├── keywords (text[])       — كلمات بحث: ["خبر","إضافة","نشر","مقال"]
├── content (text/markdown) — الشرح الكامل + خطوات
├── media_url (text)        — رابط صورة أو فيديو توضيحي (اختياري)
├── action_label (text)     — نص زر الإجراء: "اذهب لإضافة خبر"
├── action_url (text)       — "/admin/media-center?action=new"
├── related_ids (uuid[])    — مواضيع ذات صلة
├── sort_order, is_published, created_at, updated_at
```

- RLS: قراءة عامة لـ `authenticated` + كتابة/حذف لـ `admin` فقط.
- GRANT صحيحة، و `service_role` كامل الصلاحيات.

## واجهة إدارة الشروحات

صفحة جديدة `/admin/help-center` (للأدمن فقط) تتيح:
- إضافة/تعديل/حذف/إخفاء أي شرح.
- ترتيب الفئات والمواضيع (Drag & drop).
- رفع صور/فيديوهات توضيحية إلى bucket `site-media`.
- معاينة كيف يظهر الشرح في المساعد.

## المحتوى الأساسي (سأكتبه أنا قابل للتعديل لاحقاً)

سأهيئ ~60-80 شرحاً موزعة على هذه الفئات (نصوص فقط، الصور التوضيحية يضيفها المدير لاحقاً):

- **الرئيسية ولوحة التحكم** (4 شروحات): التنقل، KPI، الإجراءات السريعة.
- **محتوى الصفحات** (8): تعديل الأقسام، الإخفاء، الإيقونات، CTA.
- **المركز الإعلامي/الأخبار** (8): إضافة، تعديل، صور، نشر/أرشفة.
- **البرامج** (6).
- **من نحن + الأعضاء** (6).
- **الاستبيانات** (7): إنشاء، أسئلة، نشر، النتائج.
- **النماذج المخصصة** (6).
- **الصفحات المخصصة + القانونية** (6).
- **مجلس الإدارة** (3).
- **الشركاء + الشريط الرئيسي** (5).
- **التذييل** (3).
- **ملفات الحوكمة** (4).
- **الطلبات (تطوع/عضوية/رسائل/تقييمات)** (8).
- **الإعدادات + المستخدمون + قوالب البريد** (6).

كل شرح: عنوان واضح، 4-8 خطوات مرقمة بالعربية، زر إجراء يقود مباشرة للصفحة المناسبة.

## التقنيات المستخدمة (كلها مجانية)

| الغرض | المكتبة |
|---|---|
| البحث الذكي (تطابق ضبابي) | `fuse.js` |
| المحادثة (UI) | shadcn `Sheet` + مكونات مخصصة |
| التنقل بأزرار الإجراء | `react-router-dom` |
| التخزين والاستعلام | Lovable Cloud (موجود) |
| رفع الصور التوضيحية | bucket `site-media` (موجود) |
| تذكر المحادثة | `localStorage` (آخر 50 رسالة لكل مستخدم) |

**ملاحظة**: لا حاجة لأي edge function، لا API خارجي، لا مفاتيح، لا تكاليف تشغيل.

## خطوات التنفيذ بالترتيب

1. إنشاء الترحيل (جدول + RLS + GRANT).
2. بناء صفحة إدارة الشروحات (CRUD كامل بنمط `AdminPageContentPage` الموجود).
3. تعبئة قاعدة البيانات بالمحتوى الأساسي عبر `INSERT`.
4. بناء مكونات المساعد العائم.
5. ربط البحث بـ Fuse.js + بحث مباشر في الجداول الفعلية للموقع.
6. دمج الزر العائم في `AdminLayout`.
7. اختبار: فتح المساعد، البحث، الانتقال بزر إجراء، تعديل شرح من مركز المساعدة، التحقق من ظهور التعديل فوراً في المحادثة.

## ما لن يُنفّذ

- لا توليد ردود ذكية ديناميكية (لا LLM).
- لا تنفيذ إجراءات نيابة عن المستخدم (لن يضيف خبراً تلقائياً) — فقط يوجّهه ويفتح له الصفحة الصحيحة. هذا أكثر أماناً للمدير غير التقني.
- لا تكامل مع واتساب/بريد (يمكن إضافته لاحقاً).
