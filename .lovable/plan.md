# خطة العمل: تحسين المساعد + صفحتا الدعم والمساعدة

## 1) إصلاحات المساعد العائم الحالي

- **تمرير تلقائي للإجابة:** عند ظهور رد جديد، يتم التمرير لبداية الرسالة الأخيرة (وليس لأسفلها) باستخدام `scrollIntoView({ block: "start" })` على عنصر آخر رسالة.
- **زر "العودة للقائمة الرئيسية":** يُضاف تحت كل إجابة (مقال/نتائج/نص بلا نتيجة) زرّان: «🏠 القائمة الرئيسية» و«🔍 بحث جديد» — الأول يعيد عرض شبكة الأقسام، والثاني يفرّغ حقل البحث ويركّز عليه.
- **تحسين دقة المطابقة:**
  - تطبيع النص العربي قبل البحث (إزالة التشكيل، توحيد الألف/الياء/التاء المربوطة).
  - إضافة مرادفات شائعة (شعار = لوغو = هوية = براند، إعدادات = ضبط = تخصيص…).
  - رفع `threshold` إلى 0.45 + تفعيل `useExtendedSearch` + ترجيح أقوى للكلمات المفتاحية.
  - تخفيض الحد الأدنى لظهور نتائج (`minMatchCharLength: 2`).
- **توسيع قاعدة المعرفة:** إضافة ~25 مقالاً جديداً تغطي الفجوات الواضحة:
  - الشعار، الأيقونة (Favicon)، الهوية البصرية، الألوان، الخطوط.
  - إعدادات SEO، روابط التواصل، خرائط جوجل، البريد الرسمي.
  - تخصيص الهيدر/التذييل/الواتساب العائم/شريط الإمكانية الوصول.
  - إدارة المستخدمين والأدوار، النسخ الاحتياطي، رفع الملفات.

## 2) صفحة الدعم العامة `/support`

تظهر في رابط الهيدر العلوي + رابط Footer.

**الأقسام (بنفس ترتيب الظهور):**

```text
┌─────────────────────────────────────────┐
│   Hero Search (بحث بارز + اقتراحات)     │
├─────────────────────────────────────────┤
│   شبكة تصنيفات بأيقونات (Cards Grid)    │
│   [العضوية] [التطوع] [الخدمات] [تواصل]   │
├─────────────────────────────────────────┤
│   الأسئلة الشائعة (FAQ Accordion)       │
├─────────────────────────────────────────┤
│  نموذج تواصل  │  روابط سريعة (PDF/فيديو)│
└─────────────────────────────────────────┘
```

- بحث Fuse.js لحظي على عناوين الأسئلة + الإجابات + الكلمات المفتاحية، مع تمييز النص المطابق.
- نقر التصنيف يمرّر إلى الـ FAQ ويفلتر حسبه.
- نموذج التواصل يُرسل إلى جدول `contact_messages` الموجود.
- روابط سريعة قابلة للتحرير (عنوان + رابط/PDF + أيقونة).

## 3) صفحة الدعم للإدارة (نسخة `/admin/help-center` الحالية)

تبقى كما هي للمدير + ترقية بسيطة:
- يصبح بإمكان فتح «الصفحة الكاملة» من زر داخل المساعد العائم (Header الشيت).
- لا تغيير جوهري في وظائفها (CRUD لمقالات المساعد الداخلي).

## 4) لوحة تحكم المحتوى — كل شيء قابل للتحرير

صفحة جديدة `/admin/support-page` بثلاثة تبويبات:

| التبويب | يدير |
|---|---|
| إعدادات عامة | عنوان الصفحة، النص الترويجي، Placeholder للبحث |
| التصنيفات | إضافة/تعديل/حذف/إخفاء/إعادة ترتيب البطاقات (label, icon, color, link) |
| الأسئلة الشائعة | إضافة/تعديل/حذف/إخفاء/إعادة ترتيب، ربط بتصنيف، كلمات مفتاحية |
| الروابط السريعة | إضافة/تعديل/حذف/إخفاء/إعادة ترتيب (label, url, icon, type) |

كل العناصر تدعم: «نشر/إخفاء» + سحب وإفلات لإعادة الترتيب.

## نقاط فنية

**قاعدة البيانات** — 4 جداول جديدة:

```text
support_settings        (سطر واحد: عنوان، وصف، placeholder)
support_categories      (label, icon, color, link, sort_order, is_published)
support_faqs           (category_id, question, answer, keywords[], sort_order, is_published)
support_quick_links    (label, url, icon, type, sort_order, is_published)
```

سياسات RLS:
- قراءة عامة (anon) للعناصر المنشورة فقط.
- كتابة/حذف للمدير فقط (`has_role(auth.uid(), 'admin')`).
- GRANT مناسب لكل جدول.

**الملفات الجديدة:**
- `src/pages/Support.tsx` — صفحة عامة.
- `src/components/support/{SupportHero, CategoriesGrid, FAQAccordion, ContactForm, QuickLinks}.tsx`
- `src/pages/admin/AdminSupportPage.tsx` — إدارة المحتوى (CRUD مع تبويبات).
- `src/hooks/useSupportContent.ts` — جلب وعرض المحتوى.
- `src/lib/arabicNormalize.ts` — تطبيع النص + المرادفات (يستخدمه المساعد والصفحة).

**الملفات المعدّلة:**
- `src/components/admin/assistant/FloatingAssistant.tsx` — تمرير + زر عودة + بحث محسّن.
- `src/data/seedHelpArticles.ts` — إضافة 25 مقالاً جديداً.
- `src/components/layout/Header.tsx` + `Footer.tsx` — رابط «الدعم والمساعدة».
- `src/components/admin/AdminLayout.tsx` — رابط في الشريط الجانبي ضمن «المحتوى».
- `src/App.tsx` — تسجيل المسارات `/support` و `/admin/support-page`.

**Seed أولي:** 6 تصنيفات + 20 سؤالاً شائعاً + 5 روابط سريعة (قابلة للتعديل من المدير لاحقاً).

**التقنيات:** نفس المكدّس الحالي (React + shadcn + Fuse.js + Lovable Cloud) — بدون أي اعتماد على نماذج ذكاء اصطناعي خارجية، وبدون أي تكلفة.
