
# خطة: أقسام ووحدات تحت كل إدارة في الهيكل التنظيمي

## 1) تحويل الإدارات الثابتة إلى محتوى محرَّر
الإدارات الأربع (البرامج، المالية، الموارد البشرية، الاتصال) ومستوياتها العليا (الجمعية العمومية، مجلس الإدارة، المدير التنفيذي) أصبحت كلها محرَّرة من اللوحة عبر:
- جدول جديد `org_structure_nodes` (JSON واحد متفرّع) في `about_content` كحقل `org_structure jsonb`.
- البنية:
  ```json
  {
    "top": [
      { "id": "...", "title_ar": "الجمعية العمومية", "title_en": "...", "desc_ar": "السلطة العليا", "desc_en": "...", "variant": "primary" }
    ],
    "departments": [
      {
        "id": "...",
        "title_ar": "الإدارة المالية", "title_en": "...",
        "desc_ar": "...", "desc_en": "...",
        "sections": [
          {
            "id": "...",
            "title_ar": "قسم الشؤون المالية", "title_en": "...",
            "desc_ar": "...", "desc_en": "...",
            "units": [
              { "id": "...", "title_ar": "وحدة الحسابات العامة", "title_en": "...", "desc_ar": "", "desc_en": "" }
            ]
          }
        ]
      }
    ]
  }
  ```
- ميغريشن واحد يضيف العمود ويعبّئ القيم الافتراضية الحالية المعروضة في `About.tsx` كي لا يتغيّر العرض قبل التحرير.

## 2) لوحة التحكم — تبويب «محتوى منظم» داخل صفحة «من نحن»
في صفحة `AdminAboutPage.tsx`، إضافة قسم «الهيكل التنظيمي» يحوي:
- **العقد العليا (الجمعية العمومية / المجلس / المدير التنفيذي)**: قائمة مع اسم ar/en + وصف قصير + لون البطاقة (primary/gold/outline).
- **الإدارات**: قائمة قابلة للطي. كل إدارة:
  - حقول: الاسم ar/en، وصف قصير ar/en.
  - أزرار: «تعديل»، «حذف»، «أعلى ↑»، «أسفل ↓»، «حذف»، **«+ إضافة قسم»**.
- **الأقسام داخل كل إدارة**: قائمة فرعية مطوية، نفس الحقول (ar/en اسم + وصف قصير) + أزرار ترتيب أعلى/أسفل + سحب وإفلات + **«+ إضافة وحدة»**.
- **الوحدات داخل كل قسم**: نفس الحقول والأزرار (ترتيب + سحب وإفلات + حذف).
- جميع المستويات تدعم: السحب والإفلات (`@dnd-kit` المستخدم في المشروع) + أزرار أعلى/أسفل.

## 3) صفحة «من نحن» العامة — العرض
تحديث قسم الهيكل التنظيمي في `src/pages/About.tsx`:
- الأعلى: العقد العليا الثلاث كما هي حالياً (نفس البطاقات + الخطوط الواصلة).
- الإدارات: شبكة 4 أعمدة (md:grid-cols-4). كل بطاقة إدارة الآن قابلة للنقر/الطي وتعرض داخلها (وفق التصميم المختار v3 «بطاقات متدرّجة»):
  - رأس البطاقة: اسم الإدارة + الوصف (كما الحالي).
  - قائمة الأقسام: كل قسم نقطة دائرية على خط عمودي متدرّج + بطاقة شفافة `bg-white/5` بحدّ خفيف، تحوي اسم القسم + وصفه القصير.
  - داخل بطاقة القسم: شبكة عمود واحد من الوحدات `bg-black/20` بنقطة `teal-400` صغيرة.
- إن لم تكن للإدارة أقسام، تُعرض بنفس مظهرها الحالي بدون أي امتداد.
- ثنائي اللغة عبر `pick(ar, en)` كما باقي الأقسام.

## ملفات تتغيَّر
- `supabase/migrations/<new>.sql` — إضافة `about_content.org_structure jsonb` + قيمة افتراضية.
- `src/pages/admin/AdminAboutPage.tsx` — قسم «الهيكل التنظيمي» الجديد (CRUD ثلاثي المستويات + ترتيب).
- مكوّن جديد `src/components/admin/about/OrgStructureEditor.tsx` (لفصل المنطق المعقّد).
- `src/pages/About.tsx` — استبدال البلوك الثابت في قسم `#structure` بقراءة `org_structure` من `about_content` وعرضه بالتصميم المعتمد.
- `src/integrations/supabase/types.ts` — يتجدّد تلقائياً بعد الميغريشن.

## ملاحظات
- لا تغييرات على الواجهة العامة لغير قسم «الهيكل التنظيمي».
- لا تغييرات على جداول `board_*` أو `board_settings.positions`.
- التحقّق: بعد الميغريشن، الصفحة العامة تعرض نفس الإدارات الأربع، ثم تجربة إضافة 3 أقسام تحت «الإدارة المالية» من اللوحة والتأكد من ظهورها كبطاقات متدرّجة داخل بطاقة الإدارة.
