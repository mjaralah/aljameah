## الملاحظتان

**1. التكرار:** في `AdminSettingsPage.tsx` يوجد ثلاثة أزرار للحفظ: زر في `AdminPageHeader` (أعلى الصفحة)، وزر داخل المحتوى أسفل البطاقات، وزر ثالث في شريط ثابت (sticky) أسفل الصفحة. هذا تكرار غير مبرر بصرياً.

**2. الحفظ التلقائي:** رأيي كمختص UX/UI:

### نعم، الحفظ التلقائي هو الأفضل لهذه الصفحة، لكن بشروط:

**لماذا مناسب هنا؟**
- صفحة إعدادات (وليست نموذج إدخال بيانات حساسة) — التعديل فيها تدريجي وقابل للتراجع.
- يقلل الحمل المعرفي (لا حاجة لتذكر الضغط على "حفظ").
- يمنع فقدان التعديلات عند الانتقال بين التبويبات أو إغلاق المتصفح.
- نمط معتمد في Notion، Linear، Figma، وإعدادات Google/Apple.

**متى يكون خطراً؟**
- إذا كان التعديل ينشر تغييراً فورياً للزوار دون مراجعة (مثل إخفاء صفحة بالخطأ).
- إذا كانت الحقول مترابطة وتحتاج التحقق من صحة المجموعة قبل الحفظ.

**التوصية الموازنة:** حفظ تلقائي **مع تأخير (debounce) ثانية ونصف**، ومؤشر حالة واضح ("جارٍ الحفظ..." / "تم الحفظ ✓ منذ ثانيتين")، **بدون** إزالة زر الحفظ نهائياً — يبقى زر واحد في الترويسة للمستخدم الذي يريد تأكيداً يدوياً، أو لإعادة المحاولة عند فشل الشبكة.

## الخطة

### 1. تنظيف الأزرار المكررة
- إبقاء زر **"حفظ"** الموجود في `AdminPageHeader` فقط (أعلى يمين الصفحة).
- حذف زر "حفظ الإعدادات" الموجود أسفل آخر بطاقة.
- حذف الشريط الثابت السفلي (`sticky bottom-0`) بزر "حفظ التغييرات".

### 2. إضافة الحفظ التلقائي
- استخدام `useEffect` مع `setTimeout` (debounce ١٥٠٠ مللي ثانية) يراقب تغيرات كائن الإعدادات `s`.
- تجاهل التحميل الأولي (عدم الحفظ عند أول تعبئة من قاعدة البيانات).
- في حال وجود حفظ يدوي قيد التنفيذ، تأجيل المحاولة التلقائية.

### 3. مؤشر حالة الحفظ (Save Status Indicator)
بجانب زر الحفظ في الترويسة:
- `idle` — لا شيء يظهر.
- `dirty` — "تغييرات غير محفوظة" (لون كهرماني خفيف).
- `saving` — "جارٍ الحفظ..." مع أيقونة `Loader2` دوارة.
- `saved` — "تم الحفظ ✓ الآن" (يتلاشى بعد 3 ثوانٍ).
- `error` — "فشل الحفظ — أعد المحاولة" مع زر إعادة محاولة.

### 4. حماية إضافية
- اعتراض `beforeunload` إذا كانت الحالة `dirty` أو `saving` لمنع فقدان تعديلات لم تُحفظ بعد.

## التفاصيل التقنية

**الملف المعدّل:** `src/pages/admin/AdminSettingsPage.tsx` فقط.

**المنطق:**
```text
state: { status: 'idle'|'dirty'|'saving'|'saved'|'error', lastSavedAt }
useEffect([s]):
  if (firstLoad) return;
  setStatus('dirty')
  timer = setTimeout(save, 1500)
  return () => clearTimeout(timer)
```

**لا تغييرات في:** قاعدة البيانات، RLS، الـ APIs، أو سلوك بقية صفحات لوحة التحكم. التغيير معزول في صفحة الإعدادات.

**ملاحظة:** إذا أعجبتك الفكرة، يمكن لاحقاً تعميم نفس النمط على بقية صفحات الإدارة (محتوى من نحن، محتوى الصفحات...) كمرحلة ثانية منفصلة.
